# Hermes Guidebook Diagram Plan Design

Date: 2026-04-18
Project: `hermes-source-reading`
Status: draft

## 1. Goal

为 `guidebook/` 下 1-9 篇正文建立一轮统一的补图方案，先让每篇至少拥有 1 张真正帮助理解的图，而不是只做装饰性插图。

这轮设计只覆盖两件事：

1. 9 篇文章各自应该补什么图
2. 第九篇为什么应该通过“总装图 + 文本精炼”来一起改善

这份设计文档**不直接进入改文和画图执行**，而是先把图的角色、类型、位置、优先级和分批策略定清楚。

## 2. Design principles

本轮补图遵循以下原则：

### 2.1 每篇至少一张图，但图必须承担解释任务

目标不是为了满足“每篇有图”的完成度指标，而是让图承担至少一种明确功能：

- 说明顺序
- 说明分层
- 说明模块关系
- 说明闭环
- 说明对照差异

如果一张图不能帮助读者更快理解文章的中心论点，就不应进入正文。

### 2.2 采用混合生产方式

默认采用两类图并行：

- **Mermaid 优先**：顺序图、闭环图、对照图、分流图、生命周期图
- **架构图 skill 优先**：总图、分层图、系统入口图、总装图

这样能兼顾推进速度和成图质量。

### 2.3 放置位置按篇决定，但整体偏开篇总图

不同文章承担的任务不同：

- 世界观型文章更适合开篇总图
- 机制解释型文章更适合中段解释图
- 对照型文章可以偏前放，让读者在进入正文前先拿到理解抓手

### 2.4 采用“先拉齐、再增强”的分层推进策略

推荐策略：

- **第一轮**：9 篇各补 1 张核心理解图
- **第二轮**：再给最值得的 3 篇补第二张图

这样可以先统一全书的视觉理解门槛，再对关键篇目做重点增强。

## 3. Recommended approaches

### Approach A — 每篇只补 1 张核心理解图

优点：
- 节奏最快
- 成本可控
- 很适合先把整组文章完成度拉齐

缺点：
- 某些篇（尤其 05 / 08 / 09）可能仍然信息密度过高
- 单图可能不够承接全部解释负担

### Approach B — 每篇 1 张总图 + 1 张解释图

优点：
- 阅读体验最好
- 文章层级感更明显

缺点：
- 工作量大幅上升
- 容易让项目重新陷入长尾
- 会打断当前“快速拉齐质量线”的节奏

### Approach C — 先每篇 1 张，再给重点篇补第 2 张

优点：
- 兼顾整体推进和重点打磨
- 最符合当前项目阶段
- 能先解决“每篇都缺视觉抓手”的共性问题

缺点：
- 需要第二轮再判断哪些篇值得加图

## 4. Recommendation

推荐 **Approach C**。

原因：

1. 当前最明确目标是“每篇至少有一张图”
2. 这 9 篇文章整体可读性已达标，不需要全量返工，只需要更强的视觉抓手
3. 第九篇之外，多数文章的问题不是结构重写，而是缺少一个能立住中心论点的图
4. 先统一，再增强，符合项目当前的节奏控制

## 5. Article-by-article diagram plan

### 01 为什么 Hermes 不是“有记忆的 agent”，而是“能持续积累自己的 agent”

- **建议图名**：Hermes 自我进化闭环总图
- **图的作用**：作为整组文章的世界观入口图，建立“当前任务 → 沉淀 → 回流 → 下一轮”的闭环认知
- **图类型**：总览闭环图
- **生产方式**：架构图 skill 优先
- **建议位置**：开篇前段
- **核心表达**：Hermes 的价值不在于“有记忆”，而在于每轮都把过去回流成未来能力

### 02 从主循环看，Hermes 与 coding agent 有什么不同

- **建议图名**：Hermes vs 普通 coding agent 主循环对照图
- **图的作用**：把两套系统放在一张图里左右对照，校正读者对“主循环”的默认理解
- **图类型**：对照图
- **生产方式**：Mermaid 优先
- **建议位置**：偏前放
- **核心表达**：差别不在有没有 loop，而在 loop 前后有没有“过去进入现在 / 现在回流未来”的结构

### 03 Hermes 到底把什么存下来了：从静态文件层看它怎样为自我进化准备长期材料

- **建议图名**：Hermes 静态材料层分层图
- **图的作用**：展示 Hermes 存下来的不是单一记忆，而是多层不同用途的长期材料
- **图类型**：分层架构图
- **生产方式**：架构图 skill 优先
- **建议位置**：开篇前段
- **核心表达**：Hermes 的“存下来”是一个面向未来不同用途的材料体系

### 04 为什么 Hermes 会把持久记忆直接接进主循环

- **建议图名**：持久记忆进入当前回合的路径图
- **图的作用**：说明稳定记忆怎样从文件 / snapshot 进入当前轮 system prompt 背景层
- **图类型**：路径图 / 顺序图
- **生产方式**：Mermaid 优先
- **建议位置**：正文中前段
- **核心表达**：memory 不是临时检索，而是当前回合的认知地板

### 05 session recall 为什么不是历史回放，而是按需重构过去经验

- **建议图名**：session recall 两层结构图
- **图的作用**：把“持续归档”与“按需召回”两层边界一次讲清
- **图类型**：两层结构图
- **生产方式**：Mermaid 优先
- **建议位置**：开篇或正文前段
- **核心表达**：session recall 不是每轮自动回放，而是持续归档 + 按需召回 + focused summary

### 06 为什么 skill system 才是 Hermes 最像自我进化的地方

- **建议图名**：从一次成功做法到 skill 的升级路径图
- **图的作用**：说明“做通一次 → 被抽象 → 被写成 skill → 未来复用”的升级路径
- **图类型**：生命周期图 / 升级路径图
- **生产方式**：Mermaid 优先
- **建议位置**：正文中段偏前
- **核心表达**：skill system 把偶然成功升级成可重复成功

### 07 为什么 Hermes 不只是压缩上下文，而是在整理自己的过去

- **建议图名**：上下文压缩 vs 过去整理 对照图
- **图的作用**：直接对照普通压缩与 Hermes 式历史整理的目标差异
- **图类型**：对照图
- **生产方式**：Mermaid 优先
- **建议位置**：开篇或正文前段
- **核心表达**：Hermes 的 compression 不是删历史，而是把历史改写成仍可工作的形式

### 08 为什么 Hermes 的主循环收口后还没结束：background review 才是经验沉淀的关键转折

- **建议图名**：主循环之后的二段式回流图
- **图的作用**：说明“回答完成”之后，经验如何被重新审视并分流到 memory / skill / nothing to save
- **图类型**：顺序 + 分流图
- **生产方式**：Mermaid 优先
- **建议位置**：正文中前段
- **核心表达**：对 Hermes 来说，回答完不是结束，而是经验沉淀的开始

### 09 Hermes 怎样把 memory、recall、skills、compression 接成真正的 self-evolution loop

- **建议图名**：Hermes self-evolution loop 总装图
- **图的作用**：把前 1-8 篇拆开的能力重新装回同一张图里，作为整组文章的收口图
- **图类型**：总装闭环图
- **生产方式**：架构图 skill 优先
- **建议位置**：开篇前段
- **核心表达**：Hermes 的特色不是模块多，而是模块之间形成回流

## 8. Article 09 quality issue — design implication

用户反馈第九篇的问题主要有两类：

1. 废话偏多
2. AI 味偏重

初步阅读后，判断问题并不在于大结构完全错误，而在于：

- 总论重复较多
- “模块不是并列 feature，而是 loop”这一判断反复出现
- 段落常先铺一层抽象判断，再落回具体观点，导致拖沓
- 当前文本承担了过多“总装说明”责任，本该由图承担的部分还停留在语言里

因此，第九篇后续改写的方向应是：

- 让总装图更早出现
- 删除重复的总论段落
- 缩短抽象铺垫
- 把解释负担从一部分文字迁移给图

也就是说，第九篇不是只需要“润色”，而是需要通过“总装图 + 文本精炼”一起解决问题。

## 9. Scope check

本设计文档当前只覆盖：

- 每篇补图目标
- 图类型与位置
- 分批顺序
- 第九篇改写方向的设计性判断

本设计文档**不覆盖**：

- 每张图的具体节点 / 箭头文案
- Mermaid 源码
- 架构图最终视觉风格
- 第九篇改写后的具体段落稿

这些内容应在下一步实施计划里单独展开。

## 10. Success criteria

如果这轮设计被采纳，后续实施完成后应达到：

1. 9 篇正文各至少有 1 张高价值图
2. 图与文章中心论点一一对应，不出现装饰性补图
3. 第一批四篇图能显著提升读者理解效率
4. 第九篇通过“总装图 + 精炼文本”明显降低废话感和 AI 味
5. 整组文章从“能读”提升到“有明显视觉抓手、读者更容易抓住结构”
