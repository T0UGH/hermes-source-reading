# Hermes 自我进化阅读路线图 v1

> 如果你是从 `guidebook/` 目录第一次进入这一组，建议先看仓库层总入口：`README.md`；这份文件负责主阅读入口与阅读坡度本身。

## 先给结论

如果把 Hermes 当成一个普通 agent 项目去读，你大概率会先被这些层吸走：

- `run_agent.py` 的 while loop
- `model_tools.py` 的工具调度
- `cli.py` 的交互壳
- `gateway/` 的多平台接入

这些当然重要，但它们不是 Hermes 最有辨识度、也不是最值得学的地方。

更值得学的主线其实是：

> **Hermes 怎样把一次次对话中的经验，沉淀成下一次运行可以直接使用的长期能力。**

也就是说，这次阅读不先按“通用 agent 分层”展开，而是先按 **自我进化机制** 展开。

---

## Hermes 的“自我进化”不是什么

这里先把边界压清。

Hermes 的“自我进化”不是：

- 在线更新模型权重
- 自动微调底层模型
- 通过 RL 让模型本身变聪明

Hermes 的“自我进化”更接近一种 **系统级成长**：

- 把环境和用户事实沉淀为持久记忆
- 把过往会话变成可召回的经验
- 把成功方法沉淀为 procedural skill
- 把长历史压缩成还能继续工作的上下文
- 把这些能力重新接回 agent 主循环

一句更白的话：

> **Hermes 变强，主要靠“越来越会整理和复用自己的经验”，而不是靠模型参数自己长出来。**

---

## 四条最值得先读的主线

### 一、持久记忆主线

先读：

- `tools/memory_tool.py`
- `agent/builtin_memory_provider.py`
- `agent/memory_manager.py`

这条线要回答：

1. 为什么有 `MEMORY.md` 和 `USER.md` 两套记忆
2. 为什么中途写 memory 会立刻写盘，但不会改当前会话 system prompt
3. frozen snapshot 为什么反而是好设计
4. builtin memory 和 external memory provider 怎样分层

这里最值得抓住的一句是：

> **Hermes 要的是“持续积累”，但不愿意为了实时刷新记忆而破坏当前会话稳定性。**

---

### 二、会话回忆主线

再读：

- `hermes_state.py`
- `tools/session_search_tool.py`

这条线要回答：

1. 为什么 Hermes 不只保存 MEMORY.md / USER.md，还要保存完整 session transcript
2. 为什么底层选 SQLite + FTS5
3. 为什么 session recall 不是把旧对话原样贴回来，而是先检索、再总结
4. 为什么“事实记忆”和“会话回忆”必须分成两层

这里最值钱的地方是：

> **Hermes 不只是会“记住”，它还会在需要时把过去会话重构成当前真正需要的回忆。**

---

### 三、技能沉淀主线

接着读：

- `tools/skill_manager_tool.py`
- `agent/skill_commands.py`
- `agent/skill_utils.py`
- `tools/skills_tool.py`

这条线是 Hermes 最像“自我进化”的核心。

要回答：

1. 为什么 skill 不是另一种 memory
2. 为什么 memory 是 declarative，而 skill 是 procedural
3. 为什么 skill 的 create / patch / edit / delete 要做成正式工具
4. 为什么 Hermes 不满足于“记住经验”，而是要把经验写成之后可复用的操作程序

一句话压缩：

> **很多 agent 只会积累事实；Hermes 进一步把“怎么做”也沉淀下来。**

---

### 四、上下文自优化主线

最后读：

- `agent/context_compressor.py`
- `agent/prompt_caching.py`
- `agent/trajectory.py`

这条线要回答：

1. 为什么长期运行的 agent 必须会压缩上下文
2. Hermes 的 compaction 为什么不是“一次性摘要”，而是支持迭代式总结更新
3. prompt caching 为什么也是长期运行能力的一部分
4. trajectory 为什么像经验资产，而不只是日志

这一层的关键不是记忆，而是：

> **Hermes 不只是积累经验，它还会主动把经验整理成更低成本、更可持续运行的形式。**

---

## 建议阅读顺序

### 路径 A：最推荐
适合：第一次系统理解 Hermes 的人。

1. `tools/memory_tool.py`
2. `agent/builtin_memory_provider.py`
3. `agent/memory_manager.py`
4. `hermes_state.py`
5. `tools/session_search_tool.py`
6. `tools/skill_manager_tool.py`
7. `agent/context_compressor.py`
8. `agent/prompt_caching.py`
9. `agent/trajectory.py`
10. `run_agent.py`

这个顺序故意把 `run_agent.py` 放在最后。

原因是：

> **先看主体 loop，你会只看到“它怎么跑”；先看 memory / skill / recall / compression，你才会看到“它为什么会越跑越像一个能积累自己的系统”。**

---

### 路径 B：只看最值钱部分
适合：你只想抓 Hermes 和别家 agent 最不一样的地方。

直接读：

1. `tools/memory_tool.py`
2. `tools/session_search_tool.py`
3. `tools/skill_manager_tool.py`
4. `agent/context_compressor.py`
5. `run_agent.py` 中 memory / skill / compressor 接入点
